# jobs throughput baseline

Generated by `just jobs-bench` (see ADR-032). Reports per-graph dispatch
overhead and throughput at three workload shapes: linear dependency
chain, perfect fan-out (no edges), and a mixed-grain graph that
interleaves short and long jobs. Reruns on every dev host; commit
results below.

The baseline is informational — not a CI gate. The Phase 3 milestone
gate is the 1 M-entity / 60 FPS single-core target tracked in
`million-entities-baseline.md` (PR 3); job throughput is a contributor
but not the headline.

## Notes

- The job graph dispatcher's static-DAG approach assumes jobs declare
  their R/W sets honestly. The R-02 oracle
  (`crates/engine-platform/tests/jobs_oracle.rs`) is the backstop —
  parallel runs at varying worker counts must produce the same final
  state digest as the single-threaded reference. If the bench shows a
  surprising flat curve at high worker counts, check the oracle first
  (a lying R/W declaration silently serialises through cache-line
  contention rather than the dependency graph).
